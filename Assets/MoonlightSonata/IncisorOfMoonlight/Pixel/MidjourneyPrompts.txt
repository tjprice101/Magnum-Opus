 INCISOR OF MOONLIGHT — PIXEL PARTICLE TEXTURES
 Midjourney Prompt Reference
================================================================================

PURPOSE:
  Very small pixel-art particle sprites used for the custom dust types in the
  Incisor of Moonlight VFX system. These are the visual representation of:

  - StarPointDust: Tiny 4-pointed star sparkles spawned along the blade edge,
    at standing-wave antinodes, during impact bursts, and in the Phase 3
    constellation starburst. Drawn at scales 0.2-0.6 with rotation and twinkle.
  - LunarMote: Crescent-shaped motes that orbit the player and blade tip.
    Drawn at scales 0.25-0.5 with orbital motion behavior.
  - ResonantPulseDust: Expanding ring particles for frequency shockwaves.
    Drawn at scales 0.15-0.3 with expansion animation.
  - General crystal shards and tuning-fork sparks from DustID types.

  These are rendered by Terraria's dust system at very small screen sizes
  (typically 4-16 pixels on screen), so they must read clearly at tiny scales.

TECHNICAL REQUIREMENTS:
  - Dimensions: 8x8, 12x12, 16x16, or 32x32 maximum
  - Art style: Clean pixel art with sharp edges, NO anti-aliasing
  - Background: Transparent (alpha channel)
  - Every pixel must be deliberate — at this scale each pixel matters
  - Must read as intended shape at 100% zoom in Terraria (very small)
  - Colors should be built from the palette but can include full brights
  - These spin, so design should look good at any rotation angle

COLOR PALETTE (Incisor of Moonlight — "The Stellar Scalpel"):
  Deep Resonance Purple:  RGB(90, 50, 160)   — #5A32A0
  Frequency Pulse Lavender: RGB(170, 140, 255) — #AA8CFF

  Crystal Edge:           RGB(220, 230, 255)  — #DCE6FF

VISUAL THEME:

  strikes, star-map geometry."

 POST-PROCESSING NOTES:
  - CRITICAL: These MUST be actual pixel art at the exact stated dimensions.
    Midjourney may generate larger images that look like pixel art — you MUST
    downscale to exact dimensions (8x8, 12x12, 16x16) using Nearest Neighbor
    resampling (NO bilinear/bicubic interpolation).
  - Better approach: Use Midjourney outputs as REFERENCE images, then
    manually recreate the sprites pixel-by-pixel in Aseprite, GraphicsGale,
    or similar pixel art tool at the target resolution.
  - Verify zero anti-aliasing: Every pixel should be a solid color with no
    semi-transparent edge pixels (except the transparent background).
  - The StarPointDust draws with rotation (RotationSpeed = 0.1-0.2f), so
    the sprite must look correct at any rotation angle.
  - The LunarMote uses orbital motion, so test the crescent shape looks
    good while spinning in a circle.
  - These are tinted at runtime via dust.color, so bright neutral colors
    (white, silver) in the sprite get tinted to any palette color.
  - File size will be extremely small (<1KB per sprite). This is correct.
  - Consider creating sprite sheets (multiple variants in one texture) for
    random variation, but the dust system expects single sprites.
