 INCISOR OF MOONLIGHT — CLEAR TRAIL TEXTURES
 Midjourney Prompt Reference
================================================================================

PURPOSE:
  Identical in concept to the Trails/ textures but specifically designed for
  OVERLAY trail rendering with fully transparent backgrounds and extremely
  soft feathered edges. These are used in the secondary glow pass of the
  CalamityStyleTrailRenderer where the trail is rendered wider and softer
  behind the primary sharp trail, creating a bloom-halo effect.

  The IncisorResonanceGlow technique in IncisorResonance.fx uses these
  textures with:
  - Gentler sine wave distortion (uDistortionAmt * 0.3)
  - Wider, softer edge fade (squared QuadraticBump = edgeFade * edgeFade)
  - Slower trail length fade (1 - coords.x * 0.85 instead of * 1.1)
  - Subtler constellation node highlights (nodeBrightness * 0.3)

  The key difference from standard Trails/ textures:
  1. Fully transparent background (NOT black — uses alpha channel)
  2. Much softer edge feathering — glow should extend nearly to the edges
  3. Lower peak brightness — these are bloom overlays, not primary trails
  4. Broader vertical spread — the glow pass renders 1.5-2x wider

TECHNICAL REQUIREMENTS:
  - Dimensions: 256x64 or 512x128 (matching Trails/ dimensions)
  - Background: FULLY TRANSPARENT (alpha = 0), NOT black
  - Format: PNG with proper alpha channel (32-bit RGBA)
  - Soft feathered edges: Alpha fades smoothly to 0 at all edges
  - Lower contrast: Peak brightness ~60-70% of Trails/ equivalents
  - Broader glow: Visible brightness extends further from center
  - Must survive UV distortion with softer shader parameters
  - Premultiplied alpha compatible for overlay blending

COLOR PALETTE (Incisor of Moonlight — "The Stellar Scalpel"):
  Deep Resonance Purple:  RGB(90, 50, 160)   — #5A32A0
  Frequency Pulse Lavender: RGB(170, 140, 255) — #AA8CFF

 POST-PROCESSING NOTES:
  - CRITICAL: Background MUST be fully transparent (alpha = 0), NOT black.
    If Midjourney generates on black background, you must manually:
    1. Select all black regions
    2. Convert black to transparency
    3. For semi-dark areas, convert brightness to alpha
    Recommended: Use "black background to premultiplied alpha" conversion.
  - Better workflow: Generate on black, then use "Screen to Alpha" conversion
    in Photoshop/GIMP where luminosity becomes alpha and the color channel
    is preserved.
  - Verify that the PNG is saved as 32-bit RGBA with proper alpha channel.
  - Peak brightness of the trail content should be 50-70% (not pure white)
    since these are bloom overlays rendered additively over the main trail.
  - The glow pass shader (IncisorResonanceGlowPS) applies its own softening:
      - edgeFade * edgeFade (squared falloff, much softer)
      - Wider trail fade (1.0 - coords.x * 0.85)
      - Gentle pulse: sin(uTime * 3.0 + coords.x * 4.0) * 0.06 + 0.94
    So the texture content gets additionally softened by the shader.
  - These are rendered at 1.5-2.0x the width of the main trail geometry,
    so the textures effectively appear 50-100% larger on screen.
  - Resize to exact power-of-two dimensions (256x64 or 512x128).
  - Test transparency by placing the texture over a colored background
    in an image editor to verify proper alpha channel behavior.
