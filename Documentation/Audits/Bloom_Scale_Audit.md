### ⚠ CONFUSING TWO-FILE GOTCHA

**`SoftGlow` exists in TWO locations at different sizes:**
- `SandboxLastPrism/Orbs/SoftGlow` = **512px** (used by `MagnumTextureRegistry.SoftGlow`)
- `VFX Asset Library/GlowAndBloom/SoftGlow` = **1024px** (used by direct `ModContent.Request` calls, `SPFTextures`, `LFTextures`)

This means `MagnumTextureRegistry.GetSoftGlow()` returns **512px** while direct-loading `VFX Asset Library/GlowAndBloom/SoftGlow` yields **1024px**. Files that look like they use the "same" SoftGlow actually render at different sizes.

| Effect Purpose | Target Rendered Size | Scale for 2160px | Scale for 1024px | Scale for 512px |
|---|---|---|---|---|
| Projectile body glow | 40–80px | 0.02–0.035 | 0.04–0.08 | 0.08–0.16 |
| Tip accent | 20–40px | 0.01–0.02 | 0.02–0.04 | 0.04–0.08 |
| Weapon swing aura | 100–150px | 0.045–0.07 | 0.1–0.15 | 0.2–0.3 |
| Big impact / AoE | 150–250px | 0.07–0.12 | 0.15–0.25 | 0.3–0.5 |
| Boss-tier supernova | 300–500px | 0.14–0.23 | 0.3–0.5 | 0.6–1.0 |

## SEVERITY LEGEND

- 🔴 **CRITICAL** — >2000px rendered. Screen-filling, guaranteed GPU stall at 4K+.
- 🟠 **SEVERE** — 1000–2000px rendered. Massive overdraw, visible lag on mid-range.
- 🟡 **HIGH** — 500–1000px rendered. Unnecessarily large, cumulative perf risk.
- ✅ **OK** — <300px rendered. Appropriate for the effect type.

## SYSTEMIC ISSUE — VFXLibrary Bloom Stack Methods

All 5 VFXLibrary files use `GetBloom()` → **PointBloom (2160px)** with multiplicative scale parameters.

**Affected files:**
- `Content/Eroica/EroicaVFXLibrary.cs` — `DrawEroicaBloomStack`, `DrawBloomSandwichLayer`
- `Content/LaCampanella/LaCampanellaVFXLibrary.cs` — same pattern
- `Content/SwanLake/SwanLakeVFXLibrary.cs` — same pattern
- `Content/Fate/FateVFXLibrary.cs` — same pattern
- `Content/EnigmaVariations/EnigmaVFXLibrary.cs` — same pattern (lower multipliers, less severe)

**The problem:** The outer layer uses `scale * 2.5f` (BloomSandwich) or `scale * 2.0f` (BloomStack). With a 2160px texture:

| Caller passes `scale` | Outer layer actual | Rendered size |
|---|---|---|
| 0.05 | 0.125 | **270px** ✅ |
| 0.10 | 0.25 | **540px** 🟡 |
| 0.15 | 0.375 | **810px** 🟡 |
| 0.20 | 0.50 | **1080px** 🟠 |
| 0.30 | 0.75 | **1620px** 🟠 |

**Recommendation:** These utility methods should either:
1. Switch to `GetSoftGlow()` (512px) — then callers' existing scales become reasonable
2. Add scale clamping: `scale = Math.Min(scale, 0.07f)` before applying multipliers
3. Document that callers must pass `scale < 0.07f` for the outer layer to stay under 300px

The **EnigmaVFXLibrary** already uses much lower multipliers (`0.20f`, `0.12f`, `0.35f * coreScale`) and is substantially less problematic.
