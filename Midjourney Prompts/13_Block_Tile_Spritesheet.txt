## TECHNICAL SPECIFICATIONS - READ FIRST!
================================================================================

Terraria tile spritesheets follow a VERY SPECIFIC format that tModLoader requires.
Understanding this format is CRITICAL for your sprites to work in-game.

### STANDARD TILE SPRITESHEET DIMENSIONS:
- **Basic Single-Variant**: 288 x 270 pixels (18 columns Ã— 15 rows of 16x16 tiles + 2px padding)
- **Multi-Variant (2x2)**: 576 x 540 pixels (for 4 texture variations)
- **Multi-Variant (3x3)**: 864 x 810 pixels (for 9 texture variations)
- **Multi-Variant (4x4)**: 1152 x 1080 pixels (for 16 texture variations)

### THE TILE FRAME GRID:
Each 16x16 tile has 2 pixels of padding, making each cell 18x18.
The spritesheet contains ALL possible tile connection states:
- 4 corner tiles (top-left, top-right, bottom-left, bottom-right)
- 4 edge tiles (top, bottom, left, right)
- 4 inner corner tiles
- 1 center tile (fully surrounded)
- 6 connector/transition tiles
- Multiple single/isolated tiles

### CALAMITY'S COMMON SPRITESHEET SIZES:
| Size | Dimensions | Purpose |
|------|------------|---------|
| 234 x 90 | Single variant ore/stone | Simple blocks |
| 324 x 90 | 2 horizontal variants | Varied bricks |
| 450 x 198 | 2x2 variants | Complex natural tiles |
| 576 x 270 | Large variant set | Detailed blocks |
| 216 x 72 | Compact single | Glass-like tiles |

## POST-PROCESSING NOTES
================================================================================

After generating in Midjourney, you will need to:

1. **Resize** to exact pixel dimensions (234x90, etc.) using NEAREST NEIGHBOR interpolation
2. **Clean up** any anti-aliased edges - Terraria requires hard pixels
3. **Verify color count** - Keep palette limited (8-16 colors typically)
4. **Check tiling** - Ensure edge tiles connect seamlessly
5. **Create glowmask** version if needed (separate PNG with only glowing parts)
6. **Test in-game** - Place tiles and verify all connection states look correct
